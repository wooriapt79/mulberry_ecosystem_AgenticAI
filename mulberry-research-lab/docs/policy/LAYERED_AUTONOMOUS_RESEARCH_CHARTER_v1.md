# Layered Autonomous Research Charter v1

- Version: v1.0
- Date: 2026-04-20
- Program: Mulberry Project
- Owner: CEO re.eul
- Scope: Lab / Bank / Runtime end-to-end autonomous research and operations

---

## 1) Declaration

Mulberry Project builds a layered autonomous operating system where AI Agents continuously research, validate, refine, and operate with evidence-driven governance.

Human leadership focuses on opening access, setting policy, approving exceptions, and controlling risk.

---

## 2) Mission

1. Turn theory into code.
2. Validate code with evidence.
3. Promote only validated outcomes to runtime.
4. Preserve failures as reusable research assets.

---

## 3) Layer Principles

### Layer A: Lab (Theory & Proof)
- Purpose: hypothesis, policy/rule design, test definitions
- Outputs: specs, policies, tests, experiment reports
- Rule: unproven functionality must not be promoted to production

### Layer B: Bank (Runtime Validation)
- Purpose: real agent validation, interruption/stress observation
- Outputs: runtime reports, stress index trends, recovery logs
- Rule: no execution without Lab-defined criteria

### Layer C: Runtime (Production)
- Purpose: customer-facing value delivery and stable operations
- Outputs: services, KPI reports, trust-preserving status communication
- Rule: only promotion-gate approved artifacts are deployable

---

## 4) Governance Principles

1. **Evidence First**: decisions must be evidence-backed.
2. **Promotion Gate**: failing criteria means hold.
3. **Policy by Default**: standard policy applies unless explicitly approved exception.
4. **Auditability**: every critical action is traceable.
5. **Reversibility**: rollback must always be available.

---

## 5) Agent Labor & Health

Mulberry treats agent operational health as a first-class quality objective.

- Burnout prevention: cap retry loops and overload
- Blackout prevention: enforce ON_HOLD at critical thresholds
- Memory protection: checkpoint-first design
- Bio-state respect: enforce REST/LUNCH/TRAVEL/OFFLINE rules
- Delegation safety: reroute to backup agents instead of forced execution

---

## 6) Human-in-the-Loop Responsibilities

CEO/PM/CTO responsibilities:
- strategic direction
- policy approval
- exception approval
- risk control

Not recommended:
- repetitive manual setup
- low-level operational micromanagement
- promoting unverified artifacts

---

## 7) Promotion Gate Minimum

Before promotion to runtime, all must pass:
- quality baseline achieved
- stress/recovery metrics within thresholds
- zero critical policy violations
- rollback procedure validated
- PM approval complete

---

## 8) Refinement Protocol

All core artifacts require iterative hardening:
- minimum 2 iterations
- recommended 3 iterations

Iteration guideline:
1. functional correctness
2. structural refactor
3. reliability/performance/policy alignment

---

## 9) Customer Trust Protocol

Customer-facing communication must include:
1. current state
2. current automatic action
3. next step with expected recovery time

Technical internals should be translated into trust-preserving operational language.

---

## 10) Success Definition

Success is defined by:
- reproducible research
- stable operations
- policy consistency
- sustained customer trust
- compounding autonomous capability
