## 핵심 요약

tobecon은 **한국 B2B 업무 환경**에서 운영된다.
글로벌 표준 도구(Gmail, Slack 등)가 아닌 **한국 로컬 도구**가 기본 스택이다.

---

## 🇰🇷 한국 특이점 3가지

### 1. 📋 Notion — 워크스페이스 / 문서 허브

| 항목 | 내용 |
|------|------|
| **역할** | 업무 문서·발주 현황·거래처 정보 관리 |
| **현재 사용** | 필그람 팀 내부 문서 공유 |
| **개발 연계** | Notion API → 발주 현황 자동 업데이트 가능 |
| **비개발자 접근** | 간호사 실장도 Notion에서 직접 확인 가능 (코드 불필요) |

**통합 방향:**
```
tobecon 발주 처리
     ↓
Railway DB (SQL) 업데이트
     ↓
Notion API → 페이지 자동 갱신 (발주 상태·입고 현황)
     ↓
실장 → Notion에서 실시간 확인
```

---

### 2. 💬 카카오톡 — 주력 커뮤니케이션 채널

| 항목 | 내용 |
|------|------|
| **역할** | 팀 내부 알림 + 거래처(30개 제조사) 소통 |
| **현재 사용** | 발주 문의·확인 대부분 카카오톡으로 처리 |
| **개발 연계** | KakaoTalk Business API (알림톡) |
| **장점** | 비개발자 거래처도 카카오톡은 100% 사용 가능 |

**통합 방향:**
```
발주 처리 완료
     ↓
KakaoTalk 알림톡 자동 발송
  → 제조사: "발주 접수 완료 / 입고 예정일 ○○"
  → 실장: "SM 입고 처리 지연 알림"
     ↓
거래처 카카오 채널에서 확인
```

**API 옵션:**
- **카카오 알림톡** (비즈니스) — 공식 템플릿 기반, 거래처 발송에 적합
- **카카오 채널** — 양방향 메시지, 고객 CS 대응
- **카카오워크** — 내부팀 협업 (선택 사항)

---

### 3. 📧 네이버 메일 — 이메일 시스템

| 항목 | 내용 |
|------|------|
| **역할** | 제조사 공식 발주 이메일 (기존 방식) |
| **현재 문제** | 개별 이메일 수동 발송 → 자동화 대상 |
| **개발 연계** | Naver SMTP / Naver Works API |
| **주의** | Gmail API 아닌 네이버 SMTP 설정 필요 |

**통합 방향:**
```
발주 데이터 → DB 저장
     ↓
네이버 메일 SMTP 자동 발송
  → 수신: 제조사 담당자 이메일
  → 내용: 품목·수량·납기일·입고 주소
     ↓
발송 이력 → DB 기록 (중복 발송 방지)
```

**SMTP 설정 (예시):**
```
Host: smtp.naver.com
Port: 587 (TLS)
Auth: 네이버 앱 비밀번호 사용
From: 필그람@naver.com
```

## 🏗️ 업데이트된 기술 스택 (한국 환경 반영)

```
기존:  GitHub → Colab → Railway
추가:  + Notion API + KakaoTalk 알림톡 + Naver SMTP
```

| 레이어 | 도구 | 역할 |
|--------|------|------|
| **소스 관리** | GitHub | Koda 개발 |
| **워크벤치** | Colab | 리팩토링 |
| **배포** | Railway | 서버 운영 |
| **DB** | SQL | 발주·거래처 데이터 |
| **문서/대시** | Notion API | 비개발자 현황 확인 |
| **알림** | KakaoTalk 알림톡 | 제조사·실장 알림 |
| **이메일** | Naver SMTP | 공식 발주 메일 |

## 비개발자(간호사 실장) 접점 설계

```
실장이 보는 화면:
  Notion 페이지 (발주 현황 자동 갱신)
  카카오톡 알림 (발주 이상 시 즉시 알림)

실장이 받는 메일:
  네이버 메일 (발주 완료 리포트, 일별/주별)

실장이 건드리지 않아도 되는 것:
  GitHub, Railway, DB — 전부 자동화
```
